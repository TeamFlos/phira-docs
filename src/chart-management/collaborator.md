# 协作者功能

**该功能于 Phira v0.7.0 引入**

协作者功能允许谱面上传者在谱面简介中提及参与制作的其他用户。被提及的用户需要确认参与后，谱面才能进入审核流程。这样做既能明确谱面的贡献者，也能避免未经授权的署名。

## 提及协作者

在谱面简介中，使用以下格式提及协作者（括号不区分全半角）：

```
@用户名 (角色)
```

**示例**：
```
@Alice
@Bob (作谱协助)
@Charlie（测试反馈）
```

保存简介时，客户端会自动将其解析并填充为完整的 `@用户名#用户ID (角色)` 格式。

## 确认流程

上传或更新谱面时，系统会自动解析简介中的协作者提及，提取用户 ID 和角色信息，并向被提及的用户发送站内消息和邮件通知。协作者点击确认链接后状态更新为"已确认"，所有协作者均确认后谱面方可进入审核流程。

若有协作者未确认，上传者需主动联系对方，或从简介中移除其提及。

## 修改与删除

更新谱面简介时，系统会重新解析协作者提及：新增的协作者会收到邀请，已确认的协作者状态保持不变，角色信息会同步更新。从简介中移除某个协作者提及后，该用户即自动失去协作者身份，不再影响审核流程（历史记录中仍会保留相关信息）。

修改简介会立即生效，无需重新上传谱面文件。协作者确认后仍可被上传者移除，但建议操作前先与对方沟通。
